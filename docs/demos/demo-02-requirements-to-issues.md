# Demo 02: Requirements to issues

## Goal

Show how `REQ-001` decomposes into implementable issues without losing acceptance criteria.

## Suggested Copilot prompts

1. "Create an epic summary from `REQ-001` and decompose it into feature, quality, tech-debt, and ADR issues."
2. "Populate the feature issue template for `POST /events`, including scope, acceptance criteria, telemetry expectations, test plan, and rollout notes."

## Issue set to walk through

- Telemetry ingestion endpoint with `POST /events`
- Threshold-based rules engine
- Work-order creation and persistence stub
- Audit logging for each decision
- Notification stub
- Quality gates and CI workflow
- Structured logging and correlation IDs follow-up
- ADR request for schema versioning
